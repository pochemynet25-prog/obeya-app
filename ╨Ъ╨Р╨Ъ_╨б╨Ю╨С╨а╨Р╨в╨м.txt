КАК ПОЛУЧИТЬ ГОТОВОЕ ПРИЛОЖЕНИЕ (.apk) — коротко

1. Создай на GitHub новый репозиторий (Repositories → New), имя любое, напр. obeya-app.
   Поставь галочку "Add a README file" и нажми Create.

2. В репозитории: Add file → Upload files.
   Распакуй архив ObeyaTasks.zip и перетащи В ОКНО ВСЁ СОДЕРЖИМОЕ папки
   (файлы settings.gradle, build.gradle, gradle.properties и папки app, .github).
   Внизу нажми Commit changes.

   ВАЖНО про папку .github: её иногда не видно в проводнике (скрытая).
   Если после загрузки в репозитории НЕТ папки .github — создай файл вручную:
   Add file → Create new file → в имя впиши:  .github/workflows/build.yml
   и вставь туда содержимое файла build.yml из архива. Commit.

3. Сборка запустится сама. Открой вкладку Actions вверху репозитория.
   Дождись зелёной галочки (5–7 минут). Если красный крестик — пришли мне скриншот.

4. Открой завершённую сборку → внизу раздел Artifacts → скачай obeya-apk.
   Внутри архива — файл app-debug.apk. Это и есть приложение.

5. Перекинь app-debug.apk на телефон филиала, открой его, разреши
   "установку из этого источника", установи.

6. Открой приложение → выбери филиал → готово.
   В настройках телефона: разреши уведомления и убери приложение из
   экономии батареи (кнопка "Настройки телефона" в самом приложении помогает).

Ссылка на твою панель уже вписана в приложение (файл Config.java).
Если будешь пере-развёртывать панель — делай "Manage deployments → карандаш →
New version", чтобы ссылка /exec НЕ менялась, иначе приложение перестанет её видеть.
