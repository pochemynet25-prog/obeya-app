ОБНОВЛЕНИЕ (пуши Firebase) — Этап 2

Заливается в КОРЕНЬ репозитория, как самый первый раз.

1. Распакуй архив.
2. GitHub -> репозиторий obeya-app -> Add file -> Upload files.
3. Перетащи в окно ВСЁ содержимое архива:
   - файлы build.gradle, gradle.properties
   - папку app
   Это перезапишет изменённые файлы, google-services.json и .github не тронет.
4. Commit changes.
5. Actions -> дождись зелёной галочки (~5-7 мин, в этот раз качается больше — Firebase).
6. Скачай новый app-debug.apk, установи на телефон поверх старого.
7. Открой приложение один раз (оно зарегистрируется в Firebase).

ПРОВЕРКА (без сервера):
8. Firebase Console -> твой проект -> слева Messaging (Обмен сообщениями)
   -> Create your first campaign / Новая кампания -> Notification.
9. Впиши любой заголовок и текст -> Send test / отправь на приложение.
10. На телефоне должно прийти уведомление, даже если приложение закрыто.
    Если пришло — Firebase работает, идём к Этапу 3 (отправка задач).
